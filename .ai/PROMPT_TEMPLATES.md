# AI 初始化提示词模板

> 使用方法：根据你的项目情况，选择【新项目】或【老项目】的模板，
> 复制后粘贴发给 AI（Antigravity / Codex CLI / Cursor 等）。
> 前提：已将 SYSTEM_PROMPT.md 内容配置到 AI 工具的全局规则中。

---

## 🆕 模板一：新项目初始化

> 适用于：项目刚建立，代码还没写或刚搭好骨架。

```markdown
# 新项目初始化

本项目刚刚创建，我已在根目录准备好了 .ai/ 模板目录。
请帮我完成项目初始化，按以下顺序执行：

## 第 1 步：填充项目规范

扫描当前项目的目录结构，然后修改 `.ai/rules/global.md`，
将模板中的所有占位符替换为真实内容：

- 项目概述：项目名称、目标、状态
- 技术栈：语言、框架、编译器、平台、通信协议、存储方式
- 系统架构：整体分层图、核心模块说明、目录结构
- 编码规范：命名规则、编码约定、禁止事项
- 第三方依赖

如果某些信息你无法从代码中推断，标记为 [待填写] 让我补充。

## 第 2 步：创建功能规划

根据项目目标，帮我在 `.ai/context/progress.md` 中规划功能拆分：

- 按开发阶段划分（阶段1、阶段2...）
- 每个阶段下列出子任务
- 每个子任务必须有"描述"和"范围"
- 全部标记为 [ ] 未开始

## 第 3 步：创建模块规范

根据架构中识别到的核心模块，从 `.ai/rules/_module_template.md` 
复制并创建对应的模块规范文件。例如：
- 如果有 UI 模块 → 创建 `rules/ui.md`
- 如果有通信模块 → 创建 `rules/serial.md`
- 如果有数据模块 → 创建 `rules/data.md`

每个模块文件填写：模块概述、架构约定、边界条件。
踩坑记录留空（后续积累）。

## 第 4 步：汇报

完成后列出你做了什么，让我逐一审核。
```

## 🔄 模板二：老项目接入

> 适用于：项目已经开发了一段时间，有代码和散装文档。

```markdown
# 老项目接入 .ai 记忆系统

本项目已经开发了一段时间，我刚在根目录创建了 .ai/ 模板目录。
请帮我将现有项目信息整理到 .ai/ 体系中。按以下顺序执行：

## 第 1 步：扫描项目现状

请先做以下调研，不要修改任何文件：

1. 扫描项目目录结构，列出主要的源代码目录和文件
2. 扫描项目中所有已有的 .md 文档（不含 .ai/ 目录），
   列出每个文件的路径和内容摘要（一句话）
3. 识别项目的技术栈（语言、框架、构建工具等）
4. 识别核心模块和它们之间的依赖关系

将调研结果汇报给我，等我确认后再继续。

## 第 2 步：填充项目规范（等我确认第1步后）

根据扫描结果，修改 `.ai/rules/global.md`：
- 项目概述、技术栈、系统架构、编码规范、第三方依赖
- 把模板中的占位符替换为真实内容
- 无法确定的标记为 [待确认]

## 第 3 步：生成功能清单（等我确认第2步后）

根据代码现状，修改 `.ai/context/progress.md`：
- 已实现的功能标记 [x] 并估算完成日期
- 正在做的标记 [/]
- 还没做的标记 [ ]
- 每个子任务写上"描述"和"范围"

## 第 4 步：迁移旧文档计划（等我确认第3步后）

根据第 1 步扫描到的旧 md 文档，制定迁移计划：

| 旧文档路径 | 内容摘要 | 迁移到 .ai/ 的哪个文件 | 操作 |
|-----------|---------|----------------------|------|
| xxx.md | xxx | rules/global.md | 合并 |
| ... | ... | ... | ... |

列出计划让我确认，不要直接执行。

## 第 5 步：执行迁移（等我确认第4步后）

按照确认的迁移计划：
1. 将旧文档内容按目标文件的格式重新组织后写入
2. 不要原封不动复制，要按模板格式整理
3. 需要的话，从 _module_template.md 复制创建模块规范文件
4. 完成后列出可以归档/删除的旧文档让我确认

## 重要提醒
- 每一步完成后等我审核确认再进行下一步
- 不要跳步，不要一次性全做
- 不确定的内容标记 [待确认] 而不是猜测
```

## ⚡ 模板三：快速日常启动（每天开工用）

> 适用于：.ai/ 已经初始化好了，每天开始工作时用。

```markdown
读取 .ai/rules/global.md 和 .ai/context/active.md，
了解项目背景和当前任务状态。

如果 active.md 中有未完成的卡点，我们继续解决。
如果没有，我来告诉你今天的新任务。
```

---

## 🔀 模板四：切换新任务

> 适用于：当前任务完成，要开始下一个任务。

```markdown
当前任务已完成。请执行任务切换流程：

1. 将 .ai/context/active.md 的核心内容（任务目标、实现思路、
   边界条件、经验教训）归档追加到 .ai/logs/archive.md 顶部
2. 在 .ai/context/progress.md 中将已完成的任务打 [x] 并标注今天日期
3. 在 .ai/logs/changelog.md 顶部追加今日变更记录
4. 完成后告诉我，我来指定下一个任务
```

---

## 🐛 模板五：Bug 分析归档

> 适用于：解决了一个重大 Bug，需要记录根因分析。

```markdown
刚才解决的 Bug 很有价值，请将分析过程归档：

在 .ai/logs/archive.md 顶部追加一条技术决策记录，包含：
- 日期和标题
- 类型：Bug 根因分析
- 问题现象
- 根因分析过程
- 最终解决方案
- 经验教训

如果这个 Bug 的教训具有通用性（以后还可能遇到），
请同时建议我将经验追加到 rules/ 的哪个模块文件中。
```
